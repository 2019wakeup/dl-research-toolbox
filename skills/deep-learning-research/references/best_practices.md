# 深度学习科研最佳实践

本指南凝聚了深度学习领域的工程与科研经验，旨在提高实验效率并减少常见错误。

## 1. 实验准备与基准测试 (Baselines)
- **定义清晰的目标**：在开始任何实验前，明确要验证的假设。
- **建立基准**：始终从最简单的模型（如线性回归、基础 CNN/Transformer）开始，建立性能基准。
- **数据探索 (EDA)**：在训练前，可视化至少 50 个样本及其标签，检查数据损坏、标签错误或类别极度不平衡。

## 2. 强制性小规模测试 (Mandatory Small-scale Testing)
在大规模运行之前，必须通过以下测试：
- **过拟合单批次 (Overfit a Single Batch)**：
    - 使用 1-2 个 batch 的数据进行训练。
    - 目标：Loss 应该能迅速下降到接近 0。
    - 意义：验证模型结构、损失函数、优化器和反向传播是否跑通。
- **全流程冒烟测试 (Smoke Test)**：
    - 使用极小的数据集（如 100 个样本）。
    - 运行完整的一轮 (1 Epoch)。
    - 验证：数据增强 -> 数据加载 -> 模型前向 -> 计算 Loss -> 反向传播 -> 优化器更新 -> 指标计算 -> 日志记录 -> 模型保存。
- **梯度检查**：如果使用了自定义算子或复杂的 Loss，检查梯度是否存在梯度消失或爆炸。

## 3. 实验管理与可复现性
- **固定随机种子**：在脚本开头固定 `numpy`, `torch`, `cuda` 的种子。
- **配置管理**：使用 YAML 或 Hydra 管理超参数，严禁将超参数硬编码在代码中。
- **版本控制**：每次提交实验时，记录当前的 Git Commit ID。
- **自动化日志**：使用 WandB 或 TensorBoard 记录 Loss、指标和中间可视化结果。

## 4. 训练与验证策略
- **验证集频率**：不要等到训练结束才验证，每隔固定步数或每个 Epoch 验证一次。
- **早停 (Early Stopping)**：监控验证集指标，防止过拟合。
- **学习率调度**：优先使用带 Warmup 的 Cosine Annealing 调度器。
- **混合精度训练 (AMP)**：在显存有限或追求速度时开启，但需注意某些 Loss 可能会溢出。

## 5. 错误分析 (Error Analysis)
- **可视化失败案例**：模型预测错误的前 100 个样本是谁？它们有什么共同特征？
- **消融实验 (Ablation Study)**：系统性地移除组件，验证每个模块的实际贡献。
- **统计显著性**：对于关键结论，至少运行 3-5 次不同种子的实验，汇报均值和标准差。
